# Security Policy

## Supported Versions

This project does not provide long-term support guarantees. Security fixes are applied to the latest `main` branch.

## Reporting a Vulnerability

Please do **not** open public issues for security problems.

- Email: security@example.com (replace with a real address)
- Alternatively: open a **private** GitHub Security Advisory (if enabled) under the repo's Security tab.

Include:
- A clear description of the issue
- Steps to reproduce (if applicable)
- Impact assessment
- Any relevant logs, screenshots, or PoC details

## Disclosure Policy

We aim to acknowledge reports within 7 days and provide a remediation plan or fix timeline when possible.
